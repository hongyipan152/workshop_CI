# Security Policy

This project is used for teaching secure software engineering practices.

## Expectations
- Do not commit secrets or credentials.
- Use GitHub encrypted secrets for sensitive values.
- Pin GitHub Actions to specific versions or SHAs.
- Apply least-privilege permissions in workflows.
- Report any discovered vulnerabilities to the instructor.

## CI/CD Security Concepts Covered
- Dependency scanning
- Static analysis
- Workflow hardening
- Secret handling
- Threat modelling
